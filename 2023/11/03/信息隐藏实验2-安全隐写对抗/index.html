实验过程

LSBR

LSBR即替换LSB隐写，对于一个像素，将需要隐写的编码值直接写入该像素的最低位置。

因为总是改变像素值的最后一位，使得图像像素值的偶数往往$+1$，奇数往往$-1$，这会改变原有图像的数据分布，产生值对效应。

LSBM

为了避免值对效应的出现，LSBM将隐写规则改为：

  • 对于一个像素点，当像素值最低位与编码信息不同时，随机对该像素值进行$+1$或$-1$操作，$0$和$1$出现的概率均为$\frac{1}{2}$；
  • 若当前像素值为0，则进行$+1$操作；
  • 若当前像素值为255，则进行$-1$操作；
  • 若当前像素值最低位与编码信息相同，则不进行处理。

对于编码而言，需记录对每一个像素值的操作并返回。
对于解码而言，先根据对每一个像素值的操作，对图像进行还原处理，随后根据操作数不同，判断编码信息与原像素最低位相同还是不同。

卡方检验

隐写分析一般基于数理统计技术来分析。针对LSB隐写的分析技术有：

  • 卡方分析
  • RS分析
  • 信息量估算法
  • GPC分析法

其中，卡方分析的原理为：针对图像像素LSB全嵌入的情况，利用数理统计假设检验中的卡方分析检验模型来进行分析。

卡方检验：统计样本的实际观测值与理论推断值之间的偏离程度，决定卡方值的大小。如果卡方值越大，二者偏差程度越大；反之，二者偏差越小。若两个值完全相等时，卡方值就为0，表明理论值完全符合。

对于载密图像而言，设置图像中灰度值为$i$的像素数为$h_i$，其中$0 \leq i \leq 127$

选择一组奇偶灰度值$2i$和$2i + 1$，它们对应的像素数大小为$h_{2i}$和$h_{2i + 1}$

嵌入信息会改变直方图的分布，由差别很大变得近似相等，但是却不会改变$h_{2i} + h_{2i + 1}$的值，因为样点值要么不改变，要么就在$h_{2i}$和$h_{2i + 1}$之间改变。

构造卡方统计量$r$:

令

$$h_{2i}^* = \frac{h_{2i} + h_{2i + 1}}{2}$$

在隐写前后不会改变。

当$h_{2i}^*$较大时，根据中心极限定理有：

$$\frac{h_{2i} - h_{2i + 1}}{\sqrt{2h_{2i}^*}} = \sqrt{2} \cdot \frac{h_{2i} - h_{2i}^*}{\sqrt{h_{2i}^*}} \to N(0, 1)$$

即上式服从标准正态分布，则有：

$$r = \sum_{i = 1}^{k}{\frac{(h_{2i} - h_{2i}^*)^2}{h_{2i}^*}}$$

$r$服从卡方分布，其中$k = h_{2i - 1} + h_{2i}$

结合卡方分布的密度计算函数计算载体被隐写的概率为：

$$p = 1 - \frac{1}{2^{\frac{k-1}{2} \Gamma(\frac{k-1}{2})}} \int_{0}^{r}{exp(-\frac{t}{2})t^{\frac{k-1}{2} - 1}} ,{\rm d}t$$

如果$p$接近于$1$，则说明载体图像中含有秘密信息。

统计在不同的嵌入率下，图像的卡方统计值和$p$值的大小。

Jsteg

基本思想：将LSB嵌入法引用到量化后的DCT系数上，从而保证在隐写后人眼无法分辨隐写图与原图的差别。

嵌入方法：用秘密信息比特替换JPEG图像中量化后的DCT系数的LSB，但不处理为$0$、$1$和$-1$的DCT系数。

不使用$0$的原因：DCT系数中0的比例最大（一般在$60%$以上），而压缩编码时利用大量连续的$0$实现的；

不使用$1$和$-1$的原因：系数中$1$或$-1$若修改为$0$，接收端就无法区别是未使用的$0$还是嵌入后得到的$0$。

将原始图像AC系数的LSB替换为秘密信息bit：

  • 跳过$-1$、$0$、$1$
  • 若AC系数为$2i$，秘密比特为$0$，该系数不变
  • 若AC系数为$2i$，秘密比特为$1$，该系数变为$sign|2i + 1|$
  • 若AC系数为$2i + 1$，秘密比特为$0$，该系数变为$sign|2i|$
  • 若AC系数为$2i + 1$，秘密比特为$0$，该系数不变

嵌入步骤：

  • 将载体图像分为$8 \times 8$的块，做二维DCT变换；
  • 判断AC系数是否等于$0$或$\pm 1$，是则跳过，否则继续；
  • 提取AC系数的LSB，嵌入秘密信息。

提取步骤：

  • 将载体图像分为$8 \times 8$的块，做二维DCT变换；
  • 判断AC系数是否等于$0$或$\pm 1$，是则跳过，否则继续；
  • 提取AC系数的LSB，提取秘密信息。

根据替换规则，载密图像DCT直方图会出现值对现象。

F3

F3改进了Jsteg中的替换规则，避免了值对效应。

值为$-1$，$0$，$+1$的DCT系数再JPEG图像中占比很大，F3隐写利用了原始值为$+1$和$-1$的DCT系数，提高了嵌入率。

将原始图像AC系数的LSB替换为秘密消息bit：

  • 跳过$0$；
  • 若AC系数为$2i$，秘密比特为$0$，该系数不变；
  • 若AC系数为$2i$，秘密比特为$1$，且$i \geq 0$，该系数变为$2i - 1$；
  • 若AC系数为$2i$，秘密比特为$1$，且$i < 0$，该系数变为$2i + 1$；
  • 若AC系数为$2i + 1$，秘密比特为$1$，该系数不变；
  • 若AC系数为$2i + 1$，秘密比特为$0$，且$i \geq 0$，该系数变为$2i$；
  • 若AC系数为$2i + 1$，秘密比特为$0$，且$i < 0$，该系数变为$2i + 2$；
  • 若AC系数变为$0$，则该秘密比特需要重新隐藏。

F4

F3隐写的替换规则将会使得载密图像的DCT系数的偶系数增多。

原因：载体绝对值为1的数值较多，当其被修改为0时，嵌入算法会继续嵌入直到找到一个偶数值，或将一个奇数改为偶数。即绝对值为1的数允许嵌入1，但需要使用或制造一个偶数来嵌入0。

F4信息隐藏算法改进了F3信息隐藏算法中用到的替换规则，解决了这个问题。

F4信息隐藏算法嵌入过程遵循如下规则：

  • 跳过$0$；
  • 若AC系数为$2i$，且$i \geq 0$，秘密比特为$0$，该系数不变；
  • 若AC系数为$2i$，且$i \geq 0$，秘密比特为$1$，该系数变为$2i - 1$；
  • 若AC系数为$2i$，且$i < 0$，秘密比特为$0$，该系数变为$2i + 1$；
  • 若AC系数为$2i$，且$i < 0$，秘密比特为$1$，该系数不变；
  • 若AC系数为$2i + 1$，且$i \geq 0$，秘密比特为$0$，该系数变为$2i$；
  • 若AC系数为$2i + 1$，且$i \geq 0$，秘密比特为$1$，该系数不变；
  • 若AC系数为$2i + 1$，且$i < 0$，秘密比特为$0$，该系数不变；
  • 若AC系数为$2i + 1$，且$i < 0$，秘密比特为$1$，该系数变为$2i + 2$；
  • 若AC系数变为$0$，则该秘密比特需要重新隐藏。

载密图像的AC系数中，除$0$外，正奇数和负偶数代表1，正偶数和负奇数代表$0$。

注意！当运算后AC系数变为0时，仍然要将改变后的0值写入DCT变换域中，而不是保留原值，只是当前位置虽然做了修改，但不算有效隐藏。若保留原值，解密无法判断！

在调试中发现，在图片经历DCT逆变换再重新DCT变换后，系数值可能会发生细微变化，导致隐藏信息丢失。

F5

F5实现了基于汉明码的矩阵编码隐写，在一个分组上最多修改$R = 1$次以嵌入$2^r - 1 - r$比特，采用的基本嵌入方法为F4隐写。

实现各类统计分布特征的保持是非常困难的，因此，尽量减少对特征的扰动，就成为了安全隐写方法的首要目标。所以，我们想到了矩阵编码，借助纠错编码的相关理论和技术，通过编码方法减少每个载体分组中所需要的嵌入修改次数。

下面是一个示例：

在LSB$x_1, x_2, x_3 \in GF(2)$上，更高效嵌入$m_1, m_2$ （负载率$\frac{2}{3}$）

$$\begin{aligned}m_1 = x_1 \oplus x_3, m_2 = x_2 \oplus x_3, & 不修改 \ m_1 \neq x_1 \oplus x_3, m_2 = x_2 \oplus x_3, & 修改x_1,e_1 = 1 \ m_1 = x_1 \oplus x_3, m_2 \neq x_2 \oplus x_3, & 修改x_2,e_2 = 1 \ m_1 \neq x_1 \oplus x_3, m_2 \neq x_2 \oplus x_3, & 修改x_3,e_3 = 1 \ \end{aligned}$$

嵌入效率分析：在$3$个LSB中最多改动一次（有以上$4$种情况，平均每次在$3$个比特上修改$0.75$次）以嵌入$2$个比特信息，在负载率固定为$\frac{2}{3}$下，减少了修改次数，提高了嵌入效率，即从LSBR的$2b/次$修改提高到$2b/0.75次 = 2.67b/次$

线性变换表示：嵌入可以表达为$\bf{y} = \bf{x} + \bf{e}$，提取可以表示为线性变换

显然，$\bf{e}$的选择与$\bf{H}$也相关，这里其重量不大于$1$。

纠错码

定义：在传输过程中发生错误后能在收端自行发现或纠正的码。

编码：为使一种码具有检错或纠错能力，须对原码字增加多余的码元，以扩大码字之间的差别，即把原码字按某种规则变成有一定冗余度的码字，并使每个码字的码之间有一定的关系。关系的建立称为编码。

译码：码字到达收端后，可以根据编码规则是否满足以判定有无错误。当不能满足时，按一定规则确定错误所在位置并予以纠正。纠错并恢复原码字的过程称为译码。

将信息码首先分成若干组，然后为每组信息码附加若干位监督码元，这种编码称之为分组码。

F5嵌入步骤：

  1. 获得嵌入域。若输入的是位图，则进行JPEG编码得到JPEG系数；若输入的是JPEG图像，则进行熵编码的解码得到JPEG系数。
  2. 位置置乱。根据口令生成的密钥得到一个伪随机数发生器，偈与伪随机数发生器置乱JPEG系数的位置。
  3. 编码参数确定。为了提高嵌入效率，一般希望$n$尽可能大，因此，根据载体中可用系数的数量与消息的长度确定参数$r$，并计算$n = 2^r - 1$。此处固定$n = 7, k = 3$。
  4. 基于$(n = 7, r = 4)$汉明分组码得到矩阵编码校验矩阵，开始嵌入消息：
    • 取出n个非$0$的DCT系数，用**正奇数和负偶数代表$1$、负奇数和正偶数代表$0$**，组成一个向量$a$；
    • 取出待嵌入的k比特秘密数据，计算是否需要修改$a$，如果无需修改$(r=0)$，则返回第一步，继续下一组嵌入；
    • 如果需要修改$(r不为0)$，将要改动的DCT系数绝对值减$1$，符号不变；需要检验修改后的DCT系数为$0$：如不为$0$，则返回第一步，继续下一组嵌入; 如为$0$，则以上操作无效，删去该$0$系数，重新取出$1$个非$0$的DCT系数（原来的$n$个DCT系数中的一个变为$0$，所以新取出的DCT系数包含原来的$n−1$个系数和$1$个真正新的DCT系数），再次嵌入这$k$比特秘密数据。
  5. 位置逆置换。恢复DCT系数原来的位置顺序。
  6. 熵编码。按照JPEG标准无损压缩DCT量化系数，得到JPEG文件。

举例说明：

  • 使用$ 3 \times 7$的汉明校验矩阵$H$，在载体中嵌入秘密数据；
  • 若载体$a = (1001011)^T$，秘密数据$m = (100)^T$，计算得到$s = H \cdot a = (100)^T$，$e = s + m = (000)^T$，则$e = 0$，因此无需对$a$进行修改，$a’=a$；提取时，$m = H \cdot a = (100)^T$；
  • 若载体$a = (1001011)^T$，秘密数据$m = (011)^T$，计算得到$s = H \cdot a = (100)^T$，$e = s + m = (111)^T$，则$e = 7$，因此修改$a$中的第$7$位，$a’ = (1001010)$；提取时，$m = H \cdot a = (011)^T$；

在二进制计算中，加法即为异或运算。
