Linux编译内核并添加系统调用

内容

  • 向Linux增加一个系统调用/内核模块，鼓励程序逻辑的多样化
  • 撰写一个应用测试程序调用该系统调用/测试该内核模块
  • 使用trace/ptrace/strace，或类似的系统跟踪工具来对该测试程序进行跟踪测试

环境

  • VMware workstation 17.0.0
  • Ubuntu 20.04 64位
  • Linux内核：5.10.9

tips:

  1. 搜索你所使用的系统版本号适配哪个版本的内核，或者打开终端使用uname -r命令查看当前版本内核，选一个稍大一点的内核尝试（后者不一定保险）。
  2. 硬盘空间留大一点，60G或往上。

前期准备

1.更换国内源

  • 可视化图形界面操作法：
  1. 完成ubuntu虚拟机下载以及相关设置后，在桌面左下角找到“显示应用程序”
  2. 打开“软件与更新”
  3. 在“ubuntu软件”一栏可看到“下载自”字样，可选择“其他站点”进行修改。
  4. 推荐使用阿里云或清华云。
  • 终端修改：
  1. 查看版本号，并搜索对应的镜像源(或更改版本名)
  2. 备份原来的软件源并另存：
  3. 修改镜像源(gedit或vim)
  4. 修改完成后，更新软件列表和软件

2.下载VMware Tools

  • 点击VMware上方工作栏中的“虚拟机”
  • 点击“安装VMware Tools”（此处我已安装，显示的是重新安装）
  • 按照官方教程安装即可
  • 最后检查虚拟机页面是否对窗口大小自适应，若是，则安装成功。

编译内核所需

tips:稳定换源后此步骤更顺利哦~

获取内核源码

Linux内核官网

清华源内核文件

  • 进到下载下来的安装包所在目录下，一般将压缩包放在桌面，打开终端获取权限
  • 接下来解压缩到指定位置

添加系统调用

  • 进入内核源码解压后的位置，以下未标明操作皆在此位置下
  • 打开syscall_64.tbl文件，用于添加自定义的系统调用
  • 在最后一个common后添加自定义系统调用（此处我添加的是第441号）

添加函数声明和定义

  • 打开syscalls.h文件
  • 在末尾处添加函数声明
  • 打开sys.c文件
  • 在末尾处添加函数定义

编译内核

  • 删除无用文件
  • 配置文件，此处无需修改，直接save为.config文件并exit

可能报错1：

Your display is too small to run Menuconfig!

It must be at least 19 lines by 80 columns.

make1: *** [menuconfig] Error 1

make: *** [menuconfig] Error 2

解决方法：

该报错表示终端窗口太小，可以把终端窗口适当调大，或直接最大化即可。

  • 编译内核

可能报错2：

make1: *** 没有规则可制作目标“debian/canonical-certs.pem”，由“certs/x509_certificate_list” 需求。 停止。

解决办法：

终端使用root权限使用命令

ls -a //查看隐藏文件

gedit .config //编辑.config文件

删除以下内容（Ctrl+f快速查找）：

CONFIG_SYSTEM_TRUSTED_KEYS="debian/canonical-certs.pem" //删除双引号内的所有内容，留下双引号

重新执行make命令

可能报错3：

BTF: .tmp_vmlinux.btf: pahole (pahole) is not available

Failed to generate BTF for vmlinux
Try to disable CONFIG_DEBUG_INFO_BTF

make: *** [Makefile:1161: vmlinux] Error 1

解决办法：

一般是系统缺少dwarves软件包导致

sudo apt-get install dwarves

重新执行make命令


(跑完大概需要一个小时，咱就是说内存还是开大一点…)

  • 重置内核

可能报错4：

sh ./arch/x86/boot/install.sh 5.16.0-rc3 \arch/x86/boot/bzImage System.map “/boot”

*** Missing file: arch/x86/boot/bzImage

*** You need to run “make” before “make install”.

make: *** [arch/x86/Makefile:262：install] 错误 1

解决办法：

sudo make bzImage

可能报错5：

/bin/sh: 1: zstd: not found

make2: *** [arch/x86/boot/compressed/Makefile:139：arch/x86/boot/compressed/vmlinux.bin.zst] 错误 127

make2: *** 正在删除文件“arch/x86/boot/compressed/vmlinux.bin.zst”

make2: *** 正在等待未完成的任务…

make1: *** [arch/x86/boot/Makefile:115：arch/x86/boot/compressed/vmlinux] 错误 2

make: *** [arch/x86/Makefile:271：bzImage] 错误 2

make: *** 正在等待未完成的任务…

解决办法：

安装最新版zstd

sudo apt install aptitude

sudo aptitude install zstd

重新执行:

sudo make bzImage

sudo make modules_install

sudo make install

至此，内核的编译和安装就完成了。

  • 重启，查看当前的内核版本
  • 重启过程中按住shift选择进入“ubuntu高级选项”，选择linux-5.10.9内核，然后等待加载（稍微等一会儿，不要害怕hhh），进入界面后打开终端 如图所示为成功加载内核

验证系统调用

  • 简单写一段验证程序，输入数值“1234”.(此处直接在桌面打开终端测试即可)
  • 保存、退出，运行测试程序
  • 查看系统进程

此时，系统调用测试成功。

使用ptrace或类似工具对测试程序进行跟踪调试

strace

  • 使用strace查看程序的系统调用strace ./test
  • 可以看到此处显示syscall_0x1b9，即系统调用0x1b9(441)号，是我们刚刚添加的系统调用号
  • write是printf输出的信息
  • 使用strace -c查看测试程序的各类系统调用

gdb

  • 使用gdb调试，gdb是在ptrace的基础上得来
  • 输入l查看程序源码
  • 输入r运行程序
  • 输入b x在x行前添加断点，后先r，在用c可以一步步执行，每次到断点停止
  • 输入q退出gdb
